# CodeCore: страница регистрации (Register)

Страница регистрации CodeCore, перенесённая с макета в код. React + TypeScript + Vite. Палитра, иллюстрация и компонентная база те же, что у страницы входа (`login-codecore`), но макет hero и карточки свой.

## Превью

Двухколоночный лендинг в тёмной неоновой теме.

Слева (Hero): логотип CodeCore с тэглайном, бейдж-пилюля «Начните путь в программировании», заголовок «Создайте будущее с CodeCore» с градиентным акцентом, описание, неоновая SVG-иллюстрация по центру и нижняя панель с тремя фичами (Практические курсы, Сертификаты, Поддержка 24/7).

Справа (RegisterCard): форма регистрации с заголовком слева. В ней имя, e-mail, пароль и подтверждение пароля (оба с переключателем видимости), согласие с условиями и политикой, кнопка «Создать аккаунт», разделитель «или», регистрация через Google и ссылка на вход.

## Стек

- React 18 + TypeScript
- Vite 5
- Чистый CSS (без UI-библиотек)
- Иконки: шрифтовой набор [Flaticon UIcons](https://www.flaticon.com/ru/icon-fonts-most-downloaded), вариант Bold Rounded и Brands, подключается по CDN в `index.html`

## Запуск

```bash
npm install
npm run dev      # дев-сервер (http://localhost:5173)
npm run build    # продакшн-сборка в dist/
npm run preview  # предпросмотр сборки
```

## Интеграция с API

Слой работы с бэкендом лежит в `src/api/`. Подробные контракты с комментариями находятся прямо в коде (`client.ts`, `auth.ts`), это и есть источник правды для бэкенда.

Базовый адрес берётся из `VITE_API_BASE_URL` (см. `.env.example`), по умолчанию `/api`.

Эндпоинты, которые ожидает фронтенд:

| Метод | Путь | Назначение |
| --- | --- | --- |
| `POST` | `/auth/register` | регистрация по имени/e-mail/паролю, ставит httpOnly-cookie сессии |
| `GET` | `/auth/oauth/:provider/url` | URL для OAuth (`google` \| `github`) |

Соглашения:

- Тело запросов и ответов в формате JSON. Авторизация через httpOnly-cookie (`credentials: 'include'`).
- Ошибки отдаются как `{ message, code }` с HTTP 4xx/5xx. Поле `message` (на русском) показывается пользователю как есть.
- `POST /auth/register` возвращает `201 { user: { id, name, email, avatarUrl? } }` и `Set-Cookie`, так что после регистрации пользователь сразу авторизован.
- `409 EMAIL_TAKEN`, если e-mail уже занят; `422 VALIDATION_ERROR` при невалидных полях.
- Пароль приходит в открытом виде по HTTPS, поэтому хешируйте его на сервере (bcrypt или argon2) и продублируйте валидацию (длина ≥ 8, формат e-mail).
- Состояния запроса (loading, ошибка) `RegisterCard.tsx` обрабатывает сам: блокирует форму, меняет кнопку на «Создаём аккаунт…», показывает блок ошибки.

## Детали реализации

- Палитра и градиенты вынесены в CSS-переменные (`:root` в `index.css`).
- Градиентный текст и кнопки сделаны через `linear-gradient` + `background-clip: text`.
- Кастомные чекбоксы и кнопка «показать/скрыть пароль».
- Клиентская валидация перед запросом: совпадение паролей, минимальная длина 8, согласие с условиями. Серверную валидацию всё равно нужно продублировать.
- Карточка выше, чем у входа (4 поля и согласие), поэтому на невысоких экранах страница допускает вертикальную прокрутку (`min-height: 100vh`).
- Адаптив: при `≤980px` колонки складываются в одну, при `≤520px` соц-кнопки идут в столбик.
